# サイドミッション：Mitsuba ベース「直感で伝わる」デモ

## 位置づけ

これはメインロードマップ（`.local/roadmap.md`）の**支線**。目的は Blender 支線と同じく：

> Phase 2 までの成果を、専門知識のない人が見て
> 「透明／不透明の混ざったオブジェクトの外観がちゃんと描けてきているな」
> と直感的に感じられる絵を作る。

Blender 支線（`.local/sidemission_fancy_demo_blender.md`）との違いは主役の性格：

- **Blender**：既製アセットで“作品ショット”の見栄えに強い。ただし屈折は外表面 BSDF で補う擬似構成。
- **Mitsuba**：リポジトリで既にスクリプト化済み・完全再現可能で、
  **屈折の界面メッシュを実際に生成している**（物理により忠実）。
  一方で部屋・花瓶のような既製の見栄えアセットは乏しく、舞台はコードで組む必要がある。

本支線は「**自動・再現可能で、屈折が物理的に素直に出る検証寄りのきれいな絵**」を狙う。
係数は暫定・未校正であり、これは定性デモであって物理予測ではない旨を全出力に明示する。

## 技術的な前提

既存の `export mitsuba`（`src/vbdmat/exporters/mitsuba.py`, `docs/phase1-export-workflow.md`）は：

- ロード可能なシーン、PLY のドメイン／**IOR 界面メッシュ**、`scene-summary.json`、`capabilities.json` を生成
- `--render` で固定の EXR / 表示 PNG / 減衰診断 PNG を出力
- RGB extinction/albedo の参加媒質＋界面メッシュを使う（`g` は散乱重み付きスカラーに簡約）

つまり「向こうが歪んで透ける」に必要な屈折界面は**既にある**。
足りないのは “背景・環境光・見やすい絵作り” の部分だけ。

## マイルストーン（小さく刻む）

### M0. 現状レンダーの見やすさ改善（前提整備）
- 現行ベースラインは 256×256 / 64 spp でノイジー、背景はグレーの無地。
- まず解像度・spp を上げ、デノイザ（Mitsuba/OptiX 等、利用可能なら）を通し、
  トーンマップを整えて「ザラつきの少ない、形がはっきり見える」状態にする。
- 完了条件：window_coupon / stepped_wedge のPNGがノイズで潰れずに見える。

### M1. 「透けが分かる」ステージをコードで組む ← **AI 自動化**
- シーン生成に以下を足した `examples/phase1/demo/mitsuba_stage.py` を追加：
  - **環境光**：`envmap`（HDRI 1枚）または明るめの `constant`/エリア光。ガラスらしいハイライト用。
  - **背景プロップ**：模様のあるテクスチャ板／格子／色付きの単純ジオメトリを、
    カメラから見てオブジェクトの**真後ろ**に配置（屈折で歪むのが見える位置）。
  - **床**：接地影が出るディフューズ床。
  - カメラ・光・背景の配置を定数で固定し、再現可能に。
- 完了条件：透明サンプルを置くと、背景の模様が界面で歪んで見えることを目視確認。

### M2. VBDMAT アセットの流し込み＋高品質レンダー ← **AI 自動化**
- `export mitsuba` が出すシーン／界面メッシュ／参加媒質を M1 のステージに合成し、
  固定カメラで高解像度・デノイズありでレンダー → “きれいな絵” PNG を出力。
- 完了条件：1コマンドで両サンプルの絵が出る。

### M3. Before / After 横並び ← **AI 自動化**
- 同一ステージ・同一カメラで「プロセスモデル off（Phase 1 相当）」と
  「on（Phase 2）」を2枚レンダーし、キャプション付きで横並び比較PNGを1枚生成。
- 完了条件：普通の人が見比べて差（馴染み・にじみ・境界の変化）を言語化できる。

### M4. メインロードマップ追従の仕組み ← **AI 自動化**
- Phase 2 の各プロセスモデルを有効化した run の `optical.zarr` →
  `export mitsuba` → M2/M3 が回るドライバを用意。
- 命名規約：`.local/demo-mitsuba/<sample>/<processmodel-tag>/beauty.png`。
- 完了条件：プロセスモデルを1つ足すたび、最小コマンドでその Before/After が増える。

## 作業分担のまとめ

| 作業 | 担当 |
| --- | --- |
| レンダー品質改善（M0） | AI |
| ステージ・背景・環境光のコード実装（M1） | AI |
| アセット合成・バッチレンダー・比較生成（M2〜M4） | AI |
| HDRI／背景プロップの趣味的差し替え | 人間（任意） |

Mitsuba 支線は**基本的に AI 作業だけで完結できる**のが Blender 支線との実務的な違い。
人間の必須手作業はない（HDRI 差し替え等は任意）。

## Blender 支線との使い分け

- 「普通の人にウケる見栄え」重視 → **Blender 支線**を主役に。
- 「自動・再現可能・屈折が物理的に素直」重視 → **本 Mitsuba 支線**。
- 両立させる場合、Blender を感覚デモの表舞台、Mitsuba を物理忠実な裏取りに使うと役割が分かれる。

## 明示しておく制約（誠実さのため）

- 係数は暫定・未校正。校正済み予測ではなく**見た目の定性デモ**。
- `g` はスカラー簡約、色は RGB 近似。厳密な分光ではない。
- Blender 支線と Mitsuba のピクセルは物理的に等価ではない（媒質・界面の扱いが違う）。
  並べる際は「別レンダラの近似同士」と注記する。
